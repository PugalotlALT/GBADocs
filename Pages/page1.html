Part 1: GBA Memory

The main way to communicate with the GBA is not through software interrupts like on most OS's (although the GBA does have some). Instead, data is displayed and read by storing and loading from memory blocks and special I/O register addresses. These blocks are each designated for specific purposes which are listed below.

WRAM

Between 0x02000000 - 0x0203FFFF and 0x03000000 - 0x03007FFF is WRAM, or Working RAM. This is the block in memory which programs are able to use freely to store whatever data they so desire.

I/O Registers

Between 0x04000000 - 0x040003FE are the I/O registers or Input/Output registers. Usually 16 bits in length, these are a group of registers used for controlling the functions of the GBA such as visible screens and keypad inputs.

Palette RAM

Between 0x05000000 - 0x050003FF is the Palette RAM. This area stores the palettes, or list of colours used by sprites (we will cover more about this section later).

VRAM

Between 0x06000000 - 0x06017FFF is VRAM, or Video Memory. If you want to display something on the screen, this is the memory block you’ll need to write to, whether that be bitmap pixels or tiles (both of which we will cover later).

OAM

Between 0x07000000 - 0x070003FF is OAM, or Object Attribute Memory. This memory block stores a list of “objects”, sprites to display on the screen which can be moved and manipulated similar to objects in a traditional game engine.

There are some extra sections as well such as the Game Pak ROM but these are not necessary to know to code a basic GBA game.

Now we know about VRAM, Palette RAM and OAM, but how do these combine to display stuff on the screen? We’ll cover that in the next part. Stay tuned!
