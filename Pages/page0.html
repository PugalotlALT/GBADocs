Page 0: An Introduction

Welcome to GBADocs! This site aims to teach the fundamentals of programming for the GBA in Arm THUMB Assembly. Just so we’re all on the same page, let’s define some terms.

What is the GBA?

The GBA, or GameBoy Advance is a Nintendo console released in 2001. This was a handheld console designed to play Nintendo and officially licensed cartridges. The goal of this tutorial series is to turn our Assembly code into a working .gba file (the filetype associated with GBA ROMs, or the Read-Only Memory of a cartridge).

What is Assembly?

Computers are designed to read machine code which consists of a sequence of binary digits to represent instructions for a computer. Obviously it would be highly inconvenient for programmers to write directly in “1”s and “0”s, so we created Assembly, a language which is nearly a one-to-one conversion between machine code and more human-readable language.

Not every system has the same machine code however. Depending on the processor, different instructions might be included. For example, if you are reading this on a desktop computer running Windows or Linux, the processor in that device is probably running some version of x86 Assembly. The GBA uses the ARM7TDMI processor which does not run on x86, but rather Arm. This architecture is commonly used on devices such as mobile phones and single-board computers such as the Raspberry Pi.

For this tutorial specifically we use a subset of Arm’s instruction set called Thumb. This is a mode in the processor that reduces the instruction set to 16-bit instructions instead of the traditional 32-bit instructions of traditional Arm Assembly. This is especially beneficial for devices with limited ROM space such as the GBA. However, many of the concepts and code snippets can be edited to work with the regular Arm instruction set.

While I try to explain all the code snippets present in this tutorial, some basic understanding of Assembly is required to follow along with this guide. Some amazing resources to learn Assembly from scratch include the Putting the “You” in CPU site by HackClub or the Programmed Introduction to MIPS Assembly Language, a tutorial for another instruction set.

Tools

Throughout the tutorial I will introduce tools to do specific tasks such as sprite editing and compilation. However, there are some prerequisite tools required before starting. These include:

vasm

vasm is the compiler we will use for the duration of this tutorial. This tutorial uses “vasmarm_std_Win64.zip” specifically, but other builds may work as well. You can run this by extracting the ZIP and copying the EXE to your working directory or adding it to your path on Windows.

GBA Emulator

To run your code quickly without having to use a flash cart on a real GBA you can use a GBA emulator. The one that I primarily use and recommend is mGBA. This emulator is nearly fully accurate to the real GBA and has support for simple debugging.
